## 実機テスト結果サマリー

### テスト環境

- ハードウェア: M1 MacBook Air
- ランタイム: Ollama + Gemma 4 E2B
- プロトタイプ: `among_them_prototype.py`（Python、YAMLシナリオエンジン付き）

### シナリオ別結果

#### 1. 囚人のジレンマ（v0.3.0時点で完了済み）

- JSON出力: リトライ機構で安定化達成
- ペルソナ分化: 「裏切り傾向」「協力傾向」の差は出る
- 戦略の深み: 不足（学習・適応が見られない）
- 判定: **構造化された判断は可能。戦略性は期待できないが実験として観察可能**

#### 2. ボケて（v0.3.0時点で完了済み）

- ユーモア品質: E2Bでは不足。「説明」になり「ボケ」にならない
- 判定: **E2Bの出力品質に面白さを依存するモデルはリスクが高い → ピボットの根拠**

#### 3. AI裁判 v1 → v2

| 観察項目 | v1 | v2 |
|----------|----|----|
| 弁護士の投票 | 全ラウンド有責 | **全ラウンド無責** |
| 被告人の投票 | ラウンドで揺れる | **全ラウンド無責** |
| 投票結果 | 5-0（有責） | **3-2（有責/無責）** |
| 発言の長さ | 冗長 | やや改善 |
| 所要時間 | 1421秒（24分） | **916秒（15分、-35%）** |

- **ペルソナに【立場】【目的】を明示するとロールの一貫性が大幅に改善する** ← 最重要の知見
- 「簡潔に」指示は部分的に効果あり
- 論点のループ（ラウンド間で同じ主張）はE2Bの限界として残存

#### 4. 哲学ディベート

| 観察項目 | v1（候補未明示） | v2（候補明示） |
|----------|-----------------|---------------|
| 投票先 | 抽象ラベル（全滅） | **全員agent名で投票成功** |
| score_calc | KeyErrorでクラッシュ | **正常動作** |
| 最終結果 | N/A | 懐疑論リョウ8点、現象学ミドリ5点 |
| 所要時間 | N/A | 5598秒（93分） |

- 投票候補の明示は必須（システムプロンプトに自動挿入 + YAMLプロンプトにも記載）
- speak_each rounds × メインroundsの掛け算が致命的。5人×9周=45推論で93分
- 発言はラウンドが進むほど長くなり、無言（`...`）が発生 → コンテキスト溢れ
- 内容面: 5つの哲学的立場は方向性レベルでは分化するが、議論の深まりは乏しい

## 得られた知見（シナリオ生成プロンプトに反映済み）

1. **【立場】【目的】の明示** → ロール一貫性に明確な効果
2. **発言長の制限指示** → 部分的に効果あり
3. **ラウンド数は2〜3が適正** → 3以上で論点ループが顕著化
4. **speak_each rounds × メインrounds ≤ 50推論** → 超えると時間・品質ともに破綻
5. **vote候補の明示は必須** → 未明示だとagent名で投票できない
6. **E2Bの出力品質の限界** → 冗長さ、論点ループ、ペルソナの微妙な揺れはモデル由来。プロンプトで緩和はできるが根本解決は不可

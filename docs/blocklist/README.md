# Content Blocklist — Source of Truth

`source.json` is the authoritative source for the content blocklist. It is
**not** bundled into the app directly — the app consumes the runtime JSON at
`Pastura/Pastura/Resources/ContentBlocklist.json`, which is generated by
running `bash scripts/build-blocklist.sh`.

## Regenerating the runtime file

```
bash scripts/build-blocklist.sh
```

To verify the runtime file is in sync with this source:

```
bash scripts/build-blocklist.sh --check
```

## `contentCategory` enum values

Valid values are aligned with the OpenAI Moderation API taxonomy:

| Value        | Description                          |
|--------------|--------------------------------------|
| `harassment` | Targeted personal attacks / threats  |
| `hate`       | Slurs and discriminatory language    |
| `profanity`  | Strong expletives                    |
| `sexual`     | Sexually explicit content            |
| `violence`   | Depictions or glorification of harm  |

## Input / output policy

The `violence` category is **excluded from the input validator** — scenario
authors may write combat or conflict fiction. It is enforced on LLM output
only (ADR-005 §9.2). All other categories are checked on both input and output.
